map

"map"

resource locator of a SVG vector map to load into the ixmaps framework

"... mercator.svg" is an empty vector map to create a geographic coordinate space in mercator projection to create theme charts at geo coordinates (lat/lon) to overlay on tile maps (leaflet)

specially generated SVG vector maps can be loaded to create CHOROPLETH maps or overlay vector map layer on tile maps.

"basemap"

name of a tile map layer defined in the basemap (leaflet).

specifies the tile service for the basemap:

  • OpenStreetMap - Osmarenderer
  • OpenStreetMap - Roads
  • OpenStreetMap - Humanitarian
  • OpenStreetMap - Wikipedia
  • OpenStreetMap - Gray
  • Open Topo Map

  • Stamen - toner-lite
  • Stamen - watercolor
  • Stamen - terrain

  • CartoDB - Positron
  • CartoDB - Dark matter

  • ArcGIS - Topo
  • ArcGIS - Light Gray Base
  • ArcGIS - Ocean Basemap
  • ArcGIS - Hillshade

- scaleparam

"scaleParam": { object }

json object to define parameter related to the map scale

"normalSizeScale"

defines the normal zoom level for the adaption of chart objects to the map zoom

if "objectscaling" is set to "dynamic", the size of chart object is adapted with a dynamic growth to the map zoom level.

for this procedure it is necessary to define a zoom level for the 'normal' size of the chart object.

the level is defined by a map scale.

Example: 1:1200000 -> "1200000".

- options

"options" : { object }

parameter to control the map behavior

"featurescaling"

defines the adaption of map features to the map zoom

possible values are:

"false" map features like borders get bigger with zoom
"true" map features have always the same size
"dynamic" map features are getting bigger, but less than with "false"

"objectscaling"

defines the adaption of chart objects to the map zoom

possible values are:

"false" chart objects like pies or bar will get bigger with the map zoom
"true" chart objects will have always the same size
"dynamic" chart objects are getting bigger in a 'natural way'

if set to 'dynamic' please see the importance of the "normalsizescale" parameter

themes

"themes" : [{ array of theme objects }]

contains the definition of one or more map themes

a map theme is a data visualization defined by the parameter of one theme object

"layer"

the name of the map layer to be addressed by the theme.

obligatory for CHOROPLETH themes, mandatory for CHART themes

can be set to "generic" if the chart position is defined by geocoordinates

"field"

defines the main theme data

is the name of a data table column

the values of this column define either the color or the size of the theme objects

"$index$" is a special name to define 1 generic item per data row

"field100" : "text"

the column name of a 2° data field
which contains data for
percent/fraction/difference calculation

"style" : { style object }

defines the theme style

"type" : "key1|key2|..."

defines the type of the data visualization (theme)

alliteration of keywords to configure  chart type and data handling

"colorscheme" : [ "array of strings" ]

defines the colors of the theme

either by defining n color codes or by defining a color-scheme by generation parameter

"opacity" : "number"

defines the overall opacity of the theme from 0 to 1

"fillopacity" : "number"

defines the opacity of all area filling whithin the the theme charts from 0 to 1

"shadow" : "true or false"

defines whether or not the theme charts will be created with a shadow effect

"blur" : "number"

defines a 'blur' effect for the chart theme

"filter" : "SQL like fiter string "

defines a SQL query to apply to data table before creating the theme charts

example:

"WHERE \"Uncertainty bounds*\" = \"Median\""

"dbtable"

defines the name of the 'internal' data object.

This is the data object name you must refer in data processing procedures

It is best practice to give it a data specific name to avoid data meddling when using different data on the map

"dbtableUrl"

defines an external data source

"dbtableType"

defines the file type of the external data source

possible types: csv, json, geojson and jsonDB

"dbtableExt"

defines the name and path of an external javascript source which contains a data processor function to be executed when the theme data has been loaded

"datacache"

enables external data caching; if false, the data will be reloaded every time the internal data object is referred to in a map theme

"itemfield" : "text"

defines the data field (column) that contains position relevant data

The position data can be the id's of map shapes or geo coordinates

2 lookup fields for different columns containing latitude and longitude can be defined as one separated by '|'
example: "lookupfield":"LAT|LON"

"lookupfield" : "text"

defines the data field (column) that contains position relevant data

The position data can be the id's of map shapes or geo coordinates

2 lookup fields for different columns containing latitude and longitude must be defined as one separated by '|'
example: "lookupfield":"LAT|LON"

"showdata"

enable showing theme item data in the info window

value can be "true", or a list of data columns to show, like:

"name|country|value"

"child"

internal flag

"symbols" : [ "array of strings" ]

defines the symbols for a SYMBOL chart type

symbols can be predefined types or id's of loaded SVG symbols

pre-loaded symbols:

"circle" filled circles
"square" filled square
"label" filled rectangle
"diamond" diamond shaped symbol
"triangle" triange standing on top
"hexagon" hexagonal shape
"cross" cross symbol

"units"

defines a unit to append to the theme value display:

examples; "kg", "%"

"legendunits"

defines a unit to append to the value display in the theme legend:

examples; "kg", "%"

"normalsizevalue"

charts representing values by size are normally scaled between minimal and maximal value.

with this property you can explicitly define a value for the 'normal' size of the chart (which is about 30 pixel).

by this you are able to 'normalize' the data representation for different or realtime updated datasets.

without it, the sizes would change with different min/max values

"scale"

a relative chart scaling factor

"textcolor"

defines a color for the text parts of charts

"linecolor"

defines a color for the chart outlines

"linewidth"

defines the width of chart outlines or curve lines

"boxopacity"

define the opacity of chart BOX backgrounds

"valuescale"

scale the chart value display

"gridwidth"

the width of the aggregation grid

defined either in screen pixel ("50px") or in map distances ("1000" which equals 1km)

"aggregationscale"

an object to define different gridwidth values depending on map scales

with this, the aggregation grid can be defined in dependency of the map zoom

"rangescale"

defines a relative range scaling factor for charts

"dopacitypow"

defines the dynamic of the value based opacity of bivariate themes

it acts like a contrast factor in themes with dynamic opacity

values from 0.1 to 10

lower values = 'higher contrast'

"dopacityscale"

defines the intensity of dynamic opacity

neutral value = 1

"sizefield"

the name of the data column which values will define the chart sizes

"alphafield"

the name of the data column which values will define the chart opacity

"colorfield"

the name of the data column to select the chart color

"symbolfield"

the name of the data column to define the symbol type

"titlefield"

the name of the data column for chart titles

"valuefield"

the name of the data column which values will be showed as chart value texts

"values" : [ "array of strings" ]

array of allowed values for EXACT themes

"label" : [ "array of strings" ]

array of label to use in the theme legend

defines label for the values contained in the "values" property; for EXACT themes

"colorvalues" : [ "array of strings" ]

array of allowed values for the content of the "colorfield" data column

number of "colorvalues" must match the number of colors defined in the "colorscheme"

"symbolvalues" : [ "array of strings" ]

array of allowed values for the content of the "symbolfield" data column

number of "symbolvalues" must match the number of symbols defined in the "symbols" array

"xaxis" : [ "array of strings" ]

array of x-axis values useful for bar charts and curve plots

needs XAXIS theme key

"gridx"

defines the number of elements in x-axis direction

used in curves to define multiple curves by n x gridx values

example: defining 12 data columns for char source and gridx:3 results in 3 curves with 4 x-axis steps
